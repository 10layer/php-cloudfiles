Demonstrating how to use the PHP Cloud Files API

This is a simple web page intended to demonstrate how a Cloud Files user could use a script to "list" the contents of a Container and present links for each storage Object.

Let's assume you have a valid Cloud Files account and a Container named "IMAGES" that you use to store media files that you want to make visible to visitors. You also want to serve the files directly from Cloud Files rather store them on your web server. For this example, we'll also assume you were to use the included SAMPLE Apache configuration file to run a new Virtual Host on port 4512 (setting up the vhost is beyond the scope of this example - see the Apache Documentation for detailed instructions).

The accompanying PHP script, display.php, must be edited to store your Cloud Files user information and the $CFS_CONTAINER variable to "IMAGES". For this example, let's assume you have a picture of your dog (fido.jpg) and a video of yourself dancing in front of your web cam (disco.avi) stored in your "IMAGES" Container.

Here are links that you could put in a static HTML page (such as this one) that would use the display.php script to fetch the content from Cloud Files and stream it back to the visitor's browser:
 
This is the web page you're currently viewing.
See my dog.
Watch me dance!

  • http://your_server_name_or_ip:4512/: would see this HTML page.
  • http://your_servername_or_ip:4512/display.php?display=fido.jpg: would see the picture of your dog streamed from Cloud Files directly to the browser if it's Content-Type was set correctly when uploading into Cloud Files (no intermediate caching of the image on your server).
  • http://your_servername_or_ip:4512/display.php?display=disco.avi: would see you shake your groove-thang streamed from Cloud Files directly to the browser if it's Content-Type was set correctly when uploading into Cloud Files (no intermediate caching of the image on your server).
  • http://your_servername_or_ip:4512/display.php: with no query parameters, vistiors would see dynamically generated URLs for fido.jpg and disco.avi.
